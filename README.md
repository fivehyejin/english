## 1. 제품 개요

### 문제
영어 학원에서 매달 수십~수백 개의 예문과 패턴을 배운다. 종이 교재와 노트는 누적되면서 **"이날 뭐 배웠더라?"**가 잘 안 보인다. 찾으려 들어가도 정보가 평면적으로 나열돼 있어 핵심과 디테일이 구분이 안 된다.

### 해결
- **목차 → 날짜(Day) → 내용** 3단 구조로, 각 Day가 하나의 "노트 페이지"처럼 보이게.
- 각 Day 상단에 "이 날의 핵심"을 한눈에 파악할 수 있는 **요약 박스**.
- 수업 중 적어둔 필기(교재 주석 + 별도 종이 필기)를 원래 배운 내용과 **자연스럽게 통합**.
- 배운 동사를 **반복 체화**할 수 있는 연습 모드 (Day별·전체)

### 이 제품이 아닌 것 (명시적 비목표)
- ❌ 망각곡선 복습 관리 앱
- ❌ 플래시카드 시험 앱
- ❌ 북마크·즐겨찾기 앱
- ❌ Duolingo 같은 게이미피케이션
- ❌ SNS·공유 기능

### 사용자
혜진 (1인 사용자). 영어 회화 학원 초급 2개월차 · 10개월 Catch up 과정. 업무는 DMS UI/UX 기획. 주 사용 기기는 **iPhone (모바일 세로 모드)** + 데스크탑(Mac).

### 사용 시나리오
- **수업 후**: 앱 열어서 오늘 배운 Day를 펼쳐보며 정리
- **이동 중**: 최근 Day 스크롤하며 머릿속에 다시 훑기
- **수업 전**: 이전 Day의 "이 날의 핵심"만 빠르게 확인
- **틈틈이**: "전체 연습" 눌러서 그동안 배운 동사 짝꿍/구분/작문을 섞어서 체화

## 2. 핵심 디자인 원칙

### 원칙 1: 열자마자 '오늘 뭐 배웠는지'가 보인다
홈 화면은 Day 목차 형태. 각 Day 카드에 **주제 + 핵심 포인트 한 줄 + 그룹 개수 + 예문 수**가 바로 보여야 함.

### 원칙 2: Day 안으로 들어가면 '노트 한 페이지'처럼 쭉 보인다
- 그룹 카드 펼치기·접기 **없음**. 처음부터 다 열려있음.
- 스크롤 한 번으로 그 날 배운 모든 예문과 필기가 다 보임.
- 상단에 "이 날의 핵심" 요약 박스 (눈에 띄게).
- 수업노트는 **별도 강조 박스**로 구분 (진짜 필기처럼).

### 원칙 3: 정보에 강약이 있다
| 계층 | 시각적 처리 |
|---|---|
| **이 날의 핵심** (headline) | 페이지 상단, 큼직한 문장 |
| **Key Points** (핵심 포인트) | 불릿 리스트, 굵은 강조 |
| **수업노트 박스** | 배경색 + 아이콘 (노트 느낌) |
| **그룹 타이틀** (패턴명) | 큰 헤더 + 패턴 구조 병기 |
| **예문** | 영어(크게) → 한국어(작게) |
| **예문 필기** | 예문 아래 ✏️ 작게 |

### 원칙 4: 관리 기능은 숨긴다
복습 체크, 북마크, 플래시카드 같은 건 이 제품의 핵심이 아니다. 있어야 한다면 설정 메뉴 뒤로.

### 원칙 5: 연습은 있되 강요하지 않는다 (v2.1, v2.2)
- 홈·Day 노트 UI는 연습 모드 없이도 완결됨
- "오늘 N개 풀어야 해요" 같은 알림 없음
- 연습은 사용자가 "지금 해볼까?" 할 때만 시작
- 세션 결과·통계 누적 저장 X

## 4. 데이터 모델

자세한 타입은 `DATA_SCHEMA.md` 참고.

### 간단 요약

```typescript
interface CurriculumData {
  month: number;
  level: 'beginner' | 'intermediate';
  topic: string;
  topicNote?: string;
  daySummaries: Record<number, DaySummary>; // ← 새로 추가
  groups: PatternGroup[];
}

interface DaySummary {              // ← 새로 추가. Day 요약용
  dayTitle: string;                 // e.g. "HAVE · GET · KEEP"
  headline: string;                 // 이 날의 핵심 한 문장
  keyPoints: string[];              // 핵심 포인트 불릿 (markdown 강조 가능)
  mustRemember: string[];           // 꼭 기억할 것
}

interface PatternGroup {
  id: string;
  day: number;
  dayTopic: string;
  title: string;
  pattern?: string;
  meaning?: string;
  examples: Example[];
  kind?: 'class-note';     // 수업노트 전용 그룹
  notes?: string[];        // 그룹 레벨 필기
  noteSource?: string;     // 필기 출처 표시
}

interface Example {
  en: string;
  ko: string;
  note?: string;           // 예문별 필기
}
```

**사용자 상태는 거의 없다** (복습 관리 제거):

```typescript
interface AppState {
  lastViewedDay: { month: number; day: number } | null;  // "이어서 보기"
  theme: 'light' | 'dark';                                // 테마 설정
  practice?: {
    difficultExamples: Record<string, DifficultFlag>;     // 어려움 플래그 (v2.1)
  };
}
```

## 5. 기능 명세

### 5.1 홈 화면
- Month 정보 (topic, topicNote) 상단 고정
- 최근 본 Day 하이라이트 (`lastViewedDay` 기반, "📍 최근 학습" 배지)
- **`[🎯 전체 연습]` 버튼** (최근 학습 뱃지 옆, v2.2) — 조용한 secondary 톤
- 각 Day 카드에 `daySummary.headline` 1~2줄 + 그룹/예문 개수 표시
- Day 카드 클릭 시 Day 페이지로 이동
- Day 정렬 역순 (최신 Day가 위)

### 5.2 Day 상세 페이지
- **헤더**: Day 번호, 주제, 목차로 돌아가기, **`[📝 연습 모드]` 버튼** (v2.1)
- **이 날의 핵심 박스**: `daySummary.headline` (굵게) + `keyPoints` 리스트 + `mustRemember` (있으면 📌 아이콘 + 강조 배경)
- **수업노트 박스**: `kind === 'class-note'` 그룹들은 별도 박스로 분리해서 Day 상단 (핵심 박스 바로 아래)에 배치
  - 수업노트 박스 배경은 노트 종이 느낌 (옅은 색 + 좌측 세로선)
  - `noteSource`를 박스 타이틀에 표시 (e.g. "✏️ 수업노트 · 04/14")
- **일반 그룹들**: 순서대로 나열, 모두 펼쳐진 상태
  - 그룹 섹션 헤더: 큰 타이틀 + 패턴 + 의미
  - 그룹 레벨 필기 (`group.notes`)가 있으면 타이틀 아래 노트 박스로
  - 예문 리스트: 영어(크게 · Pretendard semibold) + 한국어(작게 · muted) + 예문 필기(✏️ 아이콘 + 옅은 배경)
  - 각 예문 우측에 🔊 발음 버튼
  - 그룹 섹션 상단에 그룹 전체 듣기 🔊 버튼 (첫 3개 예문)
- **하단 네비게이션**: 이전/다음 Day 링크

### 5.3 음성 재생 (Web Speech API)
- 예문별 🔊: 해당 영어 단독 재생
- 그룹 🔊: 첫 3개 예문 순차 재생 (`utterance.onend` 체이닝)
- `speechSynthesis.cancel()` 후 시작
- `utterance.lang = 'en-US'`, `utterance.rate = 0.9`
- 브라우저 미지원 시 `sonner` 토스트로 알림

### 5.4 라이트/다크 모드
- `next-themes` 사용
- 헤더 우측 상단 `Sun`/`Moon` 아이콘 토글
- 시스템 설정 따라가기가 기본

### 5.5 저장되는 것
- `lastViewedDay`: 마지막으로 본 Day (홈 "📍 최근 학습"에 표시용)
- `theme`: 라이트/다크 설정
- `practice.difficultExamples`: 어려움 플래그만 (v2.1부터)
- **그게 전부**. 복습 진행도·북마크·세션 기록·정답률 등 없음.

### 5.6 Day별 연습 모드 (Practice Mode) — v2.1 추가

노트를 읽는 것과 별개로, **패턴을 구분하는 감각을 키우기 위한** 별도 모드. 개별 예문을 외우는 게 아니라 **"이 상황엔 have를 써야 할까 keep일까?"** 같은 판단을 훈련.

#### 진입 방식
- Day 페이지 상단에 `[📝 연습 모드]` 버튼
- 클릭 시 `/day/[day]/practice` 이동
- Day 노트 페이지 자체는 건드리지 않음 (버튼 하나만 추가)

#### 시작 전 선택 화면

```
DAY 3 · 연습 모드

이 Day에 맞는 유형:
  ○ 빈칸 채우기 (HAVE/GET/KEEP 중 고르기)       72문제
  ○ 동사 고르기 (패턴을 먼저 인식)              72문제
  ○ 자·타동사 분류                              (Day 1 전용, 여기선 숨김)

세션 길이:
  ○ 5개   ○ 10개   ○ 전체

마지막 세션: 2일 전 · 3개 어려움 표시됨
[ 어려움 표시된 것만 풀기 (3문제) ]

[← Day 노트로]       [연습 시작 →]
```

#### 테스트 유형 (3가지)

**1. 빈칸 채우기 (Fill-in)** — 구분 학습의 핵심
```
"우린 늘 그 창문은 열어놔"

We always ___ the window open.

[ have ]  [ get ]  [ keep ]
```
- 선택지는 해당 Day의 자매 그룹에서 자동 생성
- Day 3 → have/get/keep, Day 4 → try N/try ~ing/try to V, Day 2 → could/might 등

**2. 동사 고르기 (Pattern Choice)** — 패턴 인식 먼저
```
"이 피아노 한번 쳐보세요"

어떤 try를 써야 할까요?
[ try + 명사 ]  [ try + ~ing ]  [ try + to V ]

→ 답하면 전체 문장 공개
```
- 문법 패턴을 먼저 골라야 하기 때문에 "왜 이 동사인지"를 이해시킴

**3. 자·타동사 분류 (Sort)** — Day 1 전용
```
"laugh"는 자동사? 타동사? 둘 다?

[자동사]  [타동사]  [둘 다]

힌트: 목적어를 바로 받나?
```
- Day 1에서만 제공. 다른 Day엔 숨김

#### 세션 중 화면

```
┌──────────────────────────────────────┐
│  Day 3 · 빈칸 채우기       4 / 10     │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│                                      │
│  "우린 늘 그 창문은 열어놔"             │
│                                      │
│  We always ___ the window open.      │
│                                      │
│    [ have ]   [ get ]   [ keep ]     │
│                                      │
│  [🔊 정답 듣기]                       │
│                                      │
│  ○ 어려움 표시          [다음 →]       │
└──────────────────────────────────────┘
```

- 정답 클릭 시: 초록 피드백 + 자동 🔊 재생 + 0.8초 후 다음
- 오답 클릭 시: 빨간 피드백 + 정답 강조 + `🔊 듣기` 버튼 표시 + 사용자가 "다음" 클릭해야 진행
- "어려움 표시" 체크박스: 세션 끝날 때까지 남음

#### 세션 종료 화면

```
Day 3 · 연습 완료!

맞음: 7 / 10
어려움 표시: 3개 (다음 세션에 우선 노출)

3개 틀림:
  • We always ___ the window open.    (have)
  • Keep the door ___.                 (closed)
  • Get some ___ in your face.         (wind)

[다시 하기]   [Day 노트로]
```

#### 어려움 표시 (노트에도 조용히 표시)

Day 노트 페이지에서 "어려움 표시된 예문" 왼쪽에 **조용한 점 하나**:
```
  We always have the window open.
  우린 늘 그 창문은 열어놔.                  ●
```
- 점 색: `muted-foreground` (회색), 레드 금지 — **노트 소란스러워지면 컨셉 훼손**
- 점 클릭 시 수동으로 제거 가능 (툴팁: "어려움 표시 해제")
- 다음 세션에서 맞추면 자동 제거

#### ⚠️ 절대 하지 말 것 (복습 앱 회귀 방지)

| 금지 | 대신 |
|---|---|
| 망각곡선 일정 관리 | 사용자가 원할 때만 세션 시작 |
| "오늘 할당량 N개" | 세션 길이 매번 사용자 선택 |
| 홈에 "Due 3개" 같은 뱃지 | 홈은 그대로, 연습 모드는 Day 내부에서만 |
| "마스터됨" 상태 | 어려움 플래그가 자연 소멸 (맞추면 제거) |
| 복습 스테이지 1~5단계 | 단계 없음. 플래그만 존재/비존재 |
| 성취 게이미피케이션 (streak, 점수) | 세션 끝나면 결과만 표시, 누적 X |
| 북마크 ⭐ 기능 재도입 | "어려움 표시"는 자동이지 사용자가 찍는 게 아님 |

### 5.8 전체 연습 모드 (Practice-All Mode) — v2.2 추가

Day별 연습(5.6)이 **한 Day 내 자매 그룹 구분**에 집중한다면, 전체 연습은 **지금까지 배운 모든 동사를 통합해서 반복 체화**하는 훈련장. Day 경계를 넘어 섞여 나옴.

#### 진입 방식

홈 페이지 상단 "📍 최근 학습" 뱃지 옆에 `[🎯 전체 연습]` 버튼. 클릭 시 `/practice` 이동.

홈 외의 어떤 페이지에도 흔적 없음. Day 노트 페이지, Day별 연습 모두 그대로.

#### 시작 화면

```
🎯 전체 연습

범위:
  ● 전체    ○ Day 1~3만    ○ Day 4~5만

유형 (복수 선택):
  □ 동사 짝꿍                     — 타동사 목적어 · 자동사 전치사
  □ 비슷한 동사 구분               — have/get/keep, try 3형제 등
  □ 직접 작문                     — 한국어 → 영어 타이핑

길이:
  ● 5문제   ○ 10문제   ○ 20문제

어려움 표시된 예문 8개
  → 어려움 표시된 것만 풀기

[← 홈으로]              [연습 시작 →]
```

#### 3가지 테스트 유형

##### 1. 동사 짝꿍 (Collocation) — `kind: 'collocation'`

타동사의 목적어, 자동사의 전치사와 전치사 목적어를 반복해서 체화. 데이터의 `group.pattern` 필드를 활용.

**세부 유형 세 가지를 섞어서 출제:**

**(a) 전치사만 고르기** (50%)
```
"난 부동산에 돈을 투자했어."

  I invested my money ___ real estate.
  
  [in]   [on]   [for]   [to]
```
정답 공개 시 패턴 전체 보여줌: `invest money in 투자처`.

**(b) 동사+전치사 세트 고르기** (30%, 난이도 ↑)
```
"그는 돈을 말 경주에 걸었어."

  He ___ money ___ the horse race.
  
  [invest / in]   [bet / on]   [pay / for]   [put / into]
```

**(c) 자·타동사 오류 감지** (20%, 자동사 대상만)
```
다음 문장, 맞아? 틀려?

  "She arrived the airport at 3pm."
  
  [맞다]   [틀리다]
```
정답: 틀림 (`arrive at`). 선택 시 `arrive`의 짝꿍 전치사 힌트 공개.

**대상 그룹**: Day 1의 `leave / help / prepare / pay / dance / invest / bet / walk / sing / drive`. `group.pattern` 문자열에서 전치사 추출. Day 2~5의 그룹은 이 유형에서 **제외** (짝꿍 정보가 없음).

##### 2. 비슷한 동사 구분 (Minimal Pairs) — `kind: 'minimal-pairs'`

v2.1 `fill-in`과 비슷한 구조지만 **Day 경계 없이** 자매 그룹을 묶어 출제.

```
"문을 계속 열어둬."

  ___ the door open.
  
  [have]   [get]   [keep]   [take]
```

**자매 그룹 묶음 (출제 풀):**
- **소유 4동사**: have (D3) / get (D3) / keep (D3) / take (D4) — 4지선다
- **try 3형제**: try + N (D4) / try ~ing (D4) / try to V (D4) — 3지선다
- **조동사 가능성**: will (D2) / should (D2) / can-could-might (D2) / have to (D2) — 4지선다
- **too / enough**: too + adj + to V / adj + enough + to V — 2지선다, "너무 ~해서 못해" vs "충분히 ~해서 할 수 있어" 판단

선택지는 `DATA_SCHEMA.md`의 `CROSS_DAY_SISTERS` 매핑을 사용.

##### 3. 직접 작문 (Composition) — `kind: 'composition'`

한국어 문장 보고 영어로 타이핑. 채점은 단순 비교 (LLM 채점 안 씀).

```
"그 커피는 (그녀가) 너무 뜨거워서 못 마셔."

힌트: too ~ to V 순환구조

[_______________________________________]
                                       [제출]
```

제출 후:
```
당신 답:
  The coffee is too hot to drink for her.

정답:
  The coffee is too hot (for her) to drink.
  🔊

자동 판정: △ 단어는 맞는데 순서가 다름
  (for her 위치 확인)

이번엔 어땠어?
  [✓ 맞다고 치겠어]   [✗ 다시 보겠어]
```

**채점 로직 (단순 비교)**:
1. 양쪽 문장을 소문자화 → 괄호/마침표/쉼표/물음표 제거 → 여백 정규화
2. 단어 배열로 분리
3. 판정:
   - **완벽 일치 (✓)**: 두 배열이 같음
   - **단어는 맞는데 순서 다름 (△)**: 정렬한 배열이 같음
   - **핵심 단어 일치율 ≥70% (△)**: 정답 단어의 70% 이상이 내 답에 포함
   - **그 외 (✗)**: 단어 자체가 다름
4. **최종 판정은 사용자**: `[✓ 맞다고 치겠어]` / `[✗ 다시 보겠어]` 버튼. 자동 판정은 참고용 힌트일 뿐.

**대상 예문**: 전체 예문 풀에서 랜덤. 단, 너무 긴 예문(단어 12개 초과) 제외. `group.pattern`이 있으면 힌트로 노출.

#### "어려움 표시된 것만 풀기" 모드

시작 화면에서 직접 선택. `difficultExamples`에 있는 예문만 풀. 유형은 예문이 속한 그룹 성격에 따라 자동 결정:
- Day 1 동사 그룹 예문 → collocation (pattern 추출 가능한 경우)
- 소유 4동사, try 3형제, 조동사, too/enough 예문 → minimal-pairs
- 그 외 → composition

#### 세션 중 화면

Day별 연습(5.6)의 레이아웃을 그대로 사용. 차이는 상단 라벨:

```
🎯 전체 연습 · 동사 짝꿍        3 / 10
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
...
```

#### 세션 종료 화면

Day별 연습의 결과 화면 그대로. 단 "Day N 연습 완료!" 대신 "연습 완료!".

#### 어려움 플래그 연동

v2.1 `PracticeState.difficultExamples`를 **그대로 재사용**. 스키마 변경 없음.

- Day 연습에서 표시한 어려움 → 전체 연습 "어려움만 풀기"에 나옴
- 전체 연습에서 표시한 어려움 → Day 노트 페이지에 조용한 점(●)으로 표시
- 작문 유형의 경우: 사용자가 `[✗ 다시 보겠어]` 누르면 on, `[✓ 맞다고 치겠어]` 누르면 off

#### 저장 데이터

v2.1 그대로. 신규 저장 필드 없음.

```typescript
interface PracticeState {
  difficultExamples: Record<string, DifficultFlag>;
  // 전체 연습의 세션 기록 · 정답률 · 날짜 일절 저장 안 함
}
```

#### ⚠️ 절대 하지 말 것 (복습 앱 회귀 방지, v2.1 승계)

| 금지 | 대신 |
|---|---|
| 홈에 "오늘 N개 연습하세요" 뱃지/알림 | 홈 버튼은 조용한 secondary 톤 |
| 전체 정답률 평균 누적 | 세션 결과만 보여주고 폐기 |
| Day별/유형별 진행률 게이지 | 카운트만, 퍼센트 X |
| 연속 출석(streak) 보상 | 날짜 저장 자체를 안 함 |
| 작문 정답률 랭킹 | 순위·점수 금지 |
| "이 동사는 숙달" 표시 | 어려움 플래그(있음/없음)만 유일한 상태 |
| 타이핑 중 실시간 첨삭 제안 | 제출 후 정답 공개로 끝 |
| 작문 채점용 LLM 호출 | 단순 문자열 비교만 |

## 8. 반응형

이 앱의 주 사용 환경은 **모바일 세로 모드**. 모바일 퍼스트로 설계.

### 모바일 (<768px, 기본)
- 컨테이너 `px-4 py-6`
- 홈: Day 카드 세로 스택, 한 카드 전체 너비
- Day 페이지: 한 컬럼, 텍스트 줄 길이 편하게
- 그룹 섹션 헤더는 세로 배치 (타이틀→패턴→의미)
- 예문 행도 세로 (영어→한국어)
- 🔊 버튼은 예문 아래 작게

### 태블릿 (md, ≥768px)
- 컨테이너 `px-6 py-8`
- 홈: Day 카드가 2열 그리드
- Day 페이지: 여전히 단일 컬럼 (노트처럼 읽기 편하게), 최대 너비 제한(`max-w-3xl`)

### 데스크탑 (lg, ≥1024px)
- 컨테이너 `px-8 py-10`, 최대 너비 `max-w-screen-xl mx-auto`
- Day 페이지는 `max-w-3xl` 중앙 정렬 (폭 너무 넓으면 읽기 불편)
- 예문 영어/한국어 `grid-cols-[1fr_1fr]`로 나란히

### 공통
- 폰트 크기는 본문 `text-base` 기준, 모바일에서도 작아지지 않게 (가독성 우선)
- 터치 타깃 최소 44×44px
